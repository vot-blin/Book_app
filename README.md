# Bookhouse - «Минималистичное управление персональной библиотекой»

Цель приложения: Предоставить пользователю простой и интуитивно понятный инструмент для учета прочитанных книг, планирования чтения и формирования своей цифровой библиотеки.

## Основные сущности и их атрибуты:

**Книга** — основная сущность приложения.

Обязательные атрибуты:
- **Название** (string, обязательное)  
- **Автор** (string или список строк, обязательное)  
- **Статус чтения** (перечисление):  
  - *Не начата*  
  - *В процессе*  
  - *Прочитана*  
  - *Отложена*  
- **Дата добавления** (автоматически устанавливается при создании записи)  

Опциональные атрибуты (по усмотрению пользователя, но не обязательные для ввода):
- **Год публикации** (integer)  
- **Краткое описание / аннотация** (string, необязательное)  
- **Обложка книги** (изображение или URL, опционально)

---

#### **3. Основные операции (функциональные требования)**

Пользователь может:
- **Добавлять** новую книгу, указав минимум — название и автора.  
- **Просматривать** список всех книг с возможностью фильтрации по статусу.  
- **Редактировать** информацию о книге (включая изменение статуса).  
- **Удалять** книгу из библиотеки.  
- **Помечать статус** книги (например, переключить с «Не начата» на «В процессе»).  

Дополнительно (опционально, но в рамках минимализма):
- **Поиск по названию или автору** (без сложных фильтров).  
- **Сортировка** — по названию, автору или дате добавления.

-----------------------------------------------------------------------------------------------------------------------------------------------------------

*   **Книга (Book):**
    *   *Атрибуты:* Название, автор(ы), обложка (загруженная пользователем или по ISBN), ISBN (опционально, для автоматического заполнения), жанр/теги (пользовательские), статус чтения, дата добавления.
    *   *Важно:* Информация сводится к минимуму. Нет издательства, года издания, среднего рейтинга от сообщества и т.д., если это не критично для пользователя.

*   **Коллекция (Collection/Shelf):**
    *   Виртуальные "полки" для организации книг.
    *   *Примеры:* "Хочу прочитать", "Читаю сейчас", "Прочитано", "Отложено", "Любимое", а также пользовательские коллекции ("Фэнтези", "Для учебы", "К отпуску").

*   **Прогресс чтения (Reading Progress):**
    *   *Атрибуты:* Текущая страница (или процент), общее количество страниц, дата начала чтения, дата завершения (для прочитанных книг), заметки/аннотации (приватные, только для пользователя).
    *   *Функционал:* Возможность вручную обновлять прогресс. Цель — дать ощущение движения, а не строгую статистику.

*   **Читатель (User):**
    *   Учетная запись для синхронизации библиотеки между устройствами. Личные данные минимальны (логин/email, пароль).

**3. Основные процессы (Use Cases / Сценарии):**

*   **Управление библиотекой:**
    1.  Добавление новой книги (ручной ввод, сканирование ISBN, импорт из файла).
    2.  Редактирование информации о книге (исправление опечаток, добавление своих тегов).
    3.  Удаление книги из библиотеки.
    4.  Перемещение книги между коллекциями ("полками").

*   **Процесс чтения:**
    1.  Начало чтения книги (перевод книги в статус "Читаю", установка общего количества страниц).
    2.  Обновление прогресса (ввод текущей страницы).
    3.  Добавление приватной заметки/закладки к определенной странице или главе.
    4.  Завершение чтения (отметка как "Прочитано", фиксация даты завершения).

*   **Обзор и поиск:**
    1.  Просмотр библиотеки в разных режимах (списком, сеткой, отсортировано по названию, автору, дате добавления).
    2.  Фильтрация книг по статусу ("Хочу прочитать", "Прочитано").
    3.  Поиск книги по названию или автору.
    4.  Быстрый доступ к "текущей книге" (книгам в статусе "Читаю").


**5. Ценностное предложение для пользователя:**
Приложение — это **цифровой аналог личной книжной полки и читательского дневника**. Оно помогает:
*   **Не забыть** о книгах, которые хочется прочитать.
*   **Сосредоточиться** на текущем чтении, отслеживая прогресс.
*   **Организовать** свою прочитанную литературу так, как удобно пользователю.
*   **Вести приватные заметки** по ходу чтения.
*   **Иметь быстрый доступ** ко всей своей библиотеке в одном месте, без лишнего "шума".


*   **Внешние системы (опционально):**
    *   *Облачное хранилище* (Firebase, AWS S3) — для синхронизации и резервного копирования данных пользователя.
